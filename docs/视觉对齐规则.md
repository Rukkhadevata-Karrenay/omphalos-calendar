# 月份详情页视觉对齐规则

## 素材职责

- `calendar-composition/bases/` 是正式底图，必须完整显示并使用 `object-fit: contain`。
- `calendar-composition/panels-clean/` 是覆盖底图原日期区域的正式 clean panel，禁止用 CSS blur 替代。
- `calendar-composition/date-buttons/` 是包含日期数字的正式按钮图，禁止用 CSS 或文字重绘。
- `calendar-composition/final-reference/` 只用于人工对照，禁止进入正式页面 DOM。
- 所有图片禁止裁切、拉伸、压缩、重绘和使用 `object-fit: cover`。
- 6 月正式底图原始尺寸为 1125×2436；`.month-composition-view` 与 `.month-template` 必须使用 `1125 / 2436`，不能继承旧版 1125×2350 面板比例。
- 正式底图父容器不得用 `overflow: hidden` 裁掉底部版权；clean panel 和日期层必须在版权区之前结束。

## 日期覆盖层

- 页面固定由正式底图、clean panel、日期按钮图和透明点击热区四层组成。
- 模板容器使用 `position: relative`，其他三层使用 `position: absolute`。
- 所有 `left`、`top`、`width`、`height`、横向间距和纵向间距必须从 `src/data/monthButtonLayoutConfig.ts` 读取。
- 禁止在组件或 CSS 中用随机 `margin` 调整日期位置。
- 日期按钮层与点击热区必须使用同一份网格配置，逐格完全重合。
- fiveWeek 专用 clean panel 应在素材自身 alpha 通道中完成四边渐隐，不再叠加 CSS mask；sixWeek 旧 JPG 才继续由 `src/data/monthButtonLayoutConfig.ts` 的 `panelMaskImage` 控制。
- 6 月 fiveWeek clean panel 不允许继续从旧日期区或 sixWeek panel 做整块模糊、低频采样或裁切复用；当前使用程序化二维综合色板，最终参考图只允许人工量取颜色趋势，生成脚本不得读取参考图文件。

## 日期格尺寸与状态

- 默认使用 `fiveWeek`：5 行 × 7 列，共 35 格。
- 必须保留 `sixWeek`：6 行 × 7 列，共 42 格，仅在月份确实跨 6 周时启用。
- 5 周和 6 周布局统一配置在 `src/data/monthButtonLayoutConfig.ts`，每套都必须独立包含 panel、grid、间距、行数和总格数，组件不得自行写死行数或隐藏第六行冒充五周。
- 空白日期格必须使用 `blank.jpg`，点击层使用非交互元素。
- 当前日期高亮不得改变按钮尺寸。
- 选中态、悬停态和特殊日程状态不得改变日期格尺寸或位置。
- 高亮和日程状态只能使用 outline、box-shadow 或绝对定位的独立 overlay。
- 6 月日程使用 `.date-event-frame { position: absolute; inset: 0; }` 覆盖整个日期按钮，以半透明黄紫渐变填充保留日期数字可读性；禁止缩成内部胶囊、局部色块、仅有细边框、独立长条或底部黑卡。
- 普通日期、空白格、日程格与 today/选中格必须读取同一个 `--date-cell-radius`；日期图片和日程层继承 cell 圆角，透明热区不得改变视觉形状。
- 同一月份的日期按钮不得直接复用带不同底色和纹理的逐日裁片；6 月 fiveWeek 使用 `date-buttons-normalized/06/` 的统一母版 PNG，原始 `date-buttons/06/` 只作备份与 sixWeek 备用。数字必须写入图片文件，禁止在正式页面用 CSS 或文本重画按钮。

## 当前实施范围

- 目前只启用并校准 6 月长昼月；其他月份继续沿用旧渲染。
- `.debug-layout` 代码必须保留：启用后显示 clean panel 红框、按钮层蓝框、点击热区淡黄色背景。
- 6 月正式页面默认不添加 `.debug-layout`；如需调试，只修改 `MonthCalendar.tsx` 中唯一的 debug 开关。
- 用户确认 6 月对齐后，才能将同一规则复制到其他月份。

## 验证要求

- 每次视觉修改后必须依次运行 `npm run check:assets` 和 `npm run build`。
- 重新生成 6 月 fiveWeek panel 后还必须运行 `npm run check:fiveweek-panel`，验证尺寸、alpha、五周范围、映射和源文件 hash。
- 重新生成 6 月 normalized 日期按钮后必须运行 `npm run check:date-buttons`，验证数量、尺寸、统一边缘、平均色差、原始素材 hash 与 fiveWeek/sixWeek 映射。
- 本地预览只使用 `http://localhost:5173/`，不得生成 `data:text/html` 页面或跳转到 data URL。
- 6 月当前验收检查 35 张按钮图、35 个热区、空白图、图层重合、当前日期高亮和正式 DOM 素材路径；启用 `sixWeek` 时对应检查 42 格。
